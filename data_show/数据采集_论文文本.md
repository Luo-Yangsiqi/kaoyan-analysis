# 数据采集部分（论文用文本）

## X.X 数据采集

### X.X.1 数据来源与采集概况

本研究围绕"考研热度的影响因素分析"这一核心问题，构建了涵盖被解释变量、供给侧指标、就业市场指标、替代选择指标和政策变量在内的多维数据集。原始数据来源于教育部历年统计公报、国家统计局年度数据、国家公务员局招录公告、麦可思研究院《中国大学生就业报告》（就业蓝皮书）、中国教育在线考研年度报告、百度指数官网以及多个教育数据聚合平台。由于上述数据分散于不同网站，且大多以 HTML 表格或新闻正文的形式嵌入网页之中，无统一的 API 接口可供调用，本研究采用 Python 网络爬虫技术对目标数据进行半自动采集、清洗与整合。

经过系统性的采集与整理，最终获得 2010—2024 年共 15 个年度观测点的完备面板数据，覆盖 7 大原始数据集、19 项核心指标，数据总规模约 300 余条有效记录。各数据集经合并脚本（`merge_data.py`）统一汇整为 `year.csv` 主数据表，作为后续统计建模与可视化的数据基础。

### X.X.2 爬虫技术框架与共性流程

本研究共编写了 6 个独立的数据采集脚本，分别对应不同指标的数据源。尽管各脚本因目标网页的结构差异而在解析逻辑上有所不同，但整体遵循一套统一的爬虫技术框架。本节从技术选型、反爬策略与质量控制三个维度对该共性流程加以阐述。

#### （一）爬虫框架与工具链

所有采集脚本均基于 Python 3 生态构建，核心技术栈包括：
（1）`requests` 库用于发送 HTTP 请求与管理会话；
（2）`BeautifulSoup`（bs4）库用于 HTML 文档的解析与 DOM 树遍历；
（3）`csv` 标准库用于结构化数据的本地持久化存储；
（4）`re` 正则表达式模块用于从非结构化文本中提取数值信息。

各脚本的通用执行流程可归纳为以下五个阶段：

**第一阶段：初始化与请求头配置。** 每个脚本在启动时创建全局 `requests.Session()` 会话对象，以实现连接复用和 Cookie 保持，减少重复的 TCP 握手开销。同时配置统一的 HTTP 请求头（User-Agent、Accept、Accept-Language），模拟主流浏览器（Chrome 120）的正常访问行为。

**第二阶段：多源遍历与页面抓取。** 脚本中预定义了结构化的数据源列表（`SOURCES`），每个数据源包含目标 URL、来源名称及可选的编码提示。脚本按序迭代数据源列表，调用统一的 `fetch()` 包装函数发起 GET 请求，获取页面 HTML 内容。`fetch()` 函数内置异常捕获机制（`try/except`），对网络超时、连接拒绝等异常进行容错处理，并自动识别响应的编码格式以解决部分网站返回错误字符集声明（如将 UTF-8 标注为 ISO-8859-1）的问题。

**第三阶段：多策略数据解析。** 针对不同网页的数据组织方式，脚本采用了分层次、递进式的解析策略：优先使用 HTML 表格解析（`parse_html_table`），通过对 `<table>` 元素的遍历，识别表头行中的关键词（如"年份""报名人数""录取人数"）以定位目标列，再逐行提取年份-数值对；当表格解析无法覆盖全部目标年份时，启用文本正则提取（`extract_from_text`），利用 BeautifulSoup 剔除 `<script>`、`<style>` 等非内容标签后，对页面正文应用多组正则表达式进行模糊匹配。对于特殊的数据呈现格式（如 `ticker-item` 类标签、`timeline-div` 布局），部分脚本还编写了针对性的专用解析函数。

**第四阶段：数据校验与兜底补全。** 脚本内置了来源权威的验证数据集（`VERIFIED_DATA`），将爬取结果与官方统计值进行偏差比对，对超出预设阈值（如月薪偏差 >8%、过审人数偏差 >15%）的爬取值予以拒绝并替换为验证值。对于爬虫未覆盖的年份，验证数据同时承担兜底补全功能。

**第五阶段：结果汇总与 CSV 输出。** 各脚本将采集结果按年份排序后，以 UTF-8 BOM（`utf-8-sig`）编码写入 CSV 文件，确保中文列名在 Excel 等办公软件中正确显示，并同步在控制台打印数据预览与采集统计信息。

#### （二）反爬虫策略设置

鉴于本研究的采集目标是公开的教育统计数据，目标网站大多为新闻门户或数据聚合平台，反爬机制相对宽松，但仍采取了以下基础反爬措施以确保采集过程的稳定性与合规性：

**请求头伪装。** 所有请求均携带完整的浏览器模拟请求头，包括 `User-Agent`（标识为 Windows Chrome 120）、`Accept`（接受 HTML 及 XML 格式）和 `Accept-Language`（中文优先），避免因缺失标识而被服务器拒绝服务。

**访问频率控制。** 在每个数据源抓取完成后，脚本调用 `time.sleep(2)` 设置 2 秒的请求间隔，避免短时间高频访问对目标服务器造成负担或触发速率限制。

**会话管理。** 使用 `requests.Session()` 维持持久连接，在多次请求同一域名时复用底层 TCP 连接，既提升了采集效率，也降低了对目标服务器的连接压力。

**编码兼容性处理。** 针对 Windows 环境下 GBK 终端与网页 UTF-8 编码不一致导致的输出乱码问题，各脚本统一将 `sys.stdout` 重定向为 UTF-8 编码的 `TextIOWrapper`，确保中文内容的正常显示与调试。

#### （三）数据质量控制标准

本研究在数据采集环节建立了贯穿全流程的三级质量控制体系，以确保最终入库数据的准确性与一致性：

**第一级：采集端数值范围校验。** 在 HTML 表格解析和文本正则提取阶段，所有数值均通过预设的合理区间过滤器进行初步筛查。例如，考研报名人数限定于 50—600 万、硕士录取人数限定于 20—130 万、本科毕业生数限定于 200—600 万、青年失业率限定于 5%—30%、月薪限定于 2000—15000 元。超出合理区间的提取值被视为误匹配并予以丢弃，有效排除了页面中其他无关数字（如增长率百分比、年度序号等）对目标数据的干扰。

**第二级：多源交叉验证。** 每个指标均配置了至少 2 个独立数据源。例如，国考数据通过华图教育和杭州本地宝双源交叉比对；硕士录取人数既从中国考研网的历史表格解析，又从红网、中国教育在线等新闻页面的正文中正则提取。当多源数据一致（或偏差在可接受范围内）时，以权威来源为准；当多源数据出现显著分歧时，回溯原始网页核实口径差异并进行人工研判。

**第三级：权威数据兜底与偏差修正。** 每个采集脚本均维护了一组由官方统计公报、专业研究报告等权威来源构成的验证数据集。爬取完成后，系统逐条比对爬取值与验证值：对于偏差超出阈值的记录，自动以验证值替换并输出警告日志；对于爬虫未能覆盖的年份，直接填充验证值以保证数据完整性。以月薪数据为例，爬取值与麦可思就业蓝皮书官方数据的偏差阈值设为 8%，超出该阈值的爬取值将被视为误匹配（如将高薪专业月薪误匹为全体均值）而加以修正。这一机制有效弥补了网络爬虫在非结构化数据抽取中的精度局限，使最终数据集的可靠性达到学术研究的标准。

此外，针对个别年份存在的统计口径变更问题（如 2023 年 16—24 岁青年失业率因国家统计局暂停发布旧口径数据、转而启用"不含在校生"新口径，导致该年度无完整可比均值），脚本在数据字段中以缺失值标注并附加口径说明注释，避免将不可比数据纳入后续的回归分析。
