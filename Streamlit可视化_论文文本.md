# 第 X 章  基于 Streamlit 的交互式可视化分析系统

在前述章节完成数据采集、描述统计与定量建模的基础上，本章基于 Streamlit 框架与 Plotly 交互式图表库，将研究数据与分析逻辑整合为一个轻量级 Web 可视化看板，实现对考研热度及其驱动因素的动态探索与交互展示。

---

## X.1 页面总体结构

系统采用**侧边栏—主面板**双栏布局，以年份作为贯穿全系统的交互锚点。

**侧边栏**自上而下分为三个功能区：顶部为年份范围滑块，用户可拖拽选择 2010—2024 间的任意子区间，所有模块的图表 X 轴同步缩放至该区间，切换模块时不丢失时间上下文；中部为导航单选按钮，以图标加简短标签标识六个分析模块，选中即刻切换主面板内容；底部为数据来源说明与项目信息。

**主面板**为内容展示区，各模块的图表与控件在右侧全宽渲染。每个模块内部遵循统一的信息层级：顶部为模块标题与简短说明，中部为交互式 Plotly 图表（支持悬停精确数值、框选缩放、图例过滤），图表下方设有可折叠的解读说明区，收纳方法论注释与数据口径提示。切换模块或调整侧边栏参数时，仅图表区域重新渲染，不触发整页刷新。

---

## X.2 功能模块

六个模块覆盖了从宏观概览到情景推演的分析全流程，均以时间轴为统一 X 轴基准，模块底部设有可折叠的解读说明区。

**总览看板**作为系统首页，以 PEEHI 折线图叠加低（0-33）、中（33-66）、高（66-100）三温区半透明色带作为主体图表，顶部辅以四张指标卡片——热度指数、报名人数、就业率、国考人数——卡片数值随侧边栏年份滑块实时更新，为用户提供进入系统后的第一眼宏观快照。

**就业市场分析**通过水平单选按钮在三个子视图间切换：名义 vs 实际月薪以分组柱状图对比呈现，直观展示通胀对薪资购买力的侵蚀；受雇工作比例以面积折线展示 15 年间的持续下滑轨迹，叠加均值虚线作为参考；青年失业率仅展示 2018 年及之后的有效数据，缺失年份自动跳过并在说明区注明统计制度变迁与口径断裂原因。

**考公替代效应**以双轴对比图为核心——左轴以半透明柱状展示考研报名人数，右轴以深色折线展示国考过审人数，解决了两变量数量级悬殊的视觉适配问题。下方辅以国考报录比三色分段柱状图（绿 <60 温和、黄 60-80 加剧、红 >80 高度竞争），引导用户聚焦 2023—2024 年两线首次出现的背离信号。

**招生结构分析**通过单选按钮在两张堆叠面积图间切换：学硕 vs 专硕图以蓝色与橙色面积对比展示从"蓝大橙小"到"橙大蓝小"的结构翻转，标注 2017 年专硕过半与 2020 年政策红线的关键节点；推免 vs 统考图以绿色与紫色展示推免对统考名额的持续挤压。

**综合对比**是系统中交互自由度最高的模块。用户通过多选下拉框自由勾选变量（建议 3—5 个以避免线条重叠），系统实时将所有选中变量经 Min-Max 归一化至 [0, 1] 后叠合在同一坐标系。勾选"报名人数"与"受雇工作比例"即可直观看到二者的镜像反向关系——其说服力远超静态的相关系数表格。图例支持单击隐藏/显示单条曲线，方便聚焦特定变量对。

**情景模拟**以 2024.4 处虚线为界，左侧展示历史 PEEHI 实线，右侧外推基准延续、就业压力加剧、考公分流增强与综合治理降温四条情景虚线路径。用户可通过单选按钮筛选情景组合（全部四条 / 仅基准与治理 / 仅基准、就业与考公），避免多线重叠时的视觉拥挤。图表下方附有带背景色阶的数据表格（红→黄→绿倒序），PEEHI 越高的单元格颜色越暖，扫读即可感知情景间差异。底部折叠框详细说明了模拟的假设与局限，强调其"探索性推演"而非"精确预测"的方法论定位。

## X.3 设计要点

**以时间轴为统一线索。** 本研究所有变量均为年度时序数据，因此将年份作为贯穿全系统的唯一交互锚点。用户在侧边栏调整年份范围后，从总览到情景模拟的所有图表 X 轴同步响应，避免了逐模块重设参数的重复操作。

**颜色语义一致性。** 全系统遵循固定的颜色映射规则：红色系→考研热度/PEEHI，蓝色系→薪资与就业吸引力，橙色系→就业压力，绿色系→政策供给，紫色系→考公替代。跨模块浏览时，用户无需重新学习颜色含义，降低了认知切换成本。

**渐进式信息披露。** 每个模块的信息呈现遵循"先见森林、再见树木"的认知规律：主图表展示趋势全貌（不需任何操作即可获取整体印象），悬停显示精确年份数值（按需深入），折叠说明收纳方法论细节（避免干扰主视图）。这一设计兼顾了快速浏览与深度探索的双重需求。

**探索式交互优先于固定结论。** 综合对比模块允许用户自由组合变量——勾选"报名人数"与"受雇工作比例"即可直观看到二者的镜像反向关系，其说服力远超静态的 Pearson 相关系数表格。情景模拟模块允许用户选择性屏蔽特定路径，聚焦感兴趣的对比维度。这种"让数据说话"的交互范式，将分析主动权从作者转移给了读者。
